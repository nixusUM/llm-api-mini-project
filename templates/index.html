How to Verify

1) Start a short dialog (1-2 messages) and note token metrics.
2) Continue with 8-10 turns and watch `history_tokens` + `Token Growth` table.
3) Set `Context limit tokens` to a small value (e.g. 800).
4) Send a long request and confirm overflow warning appears.
5) Compare token/cost growth between short and long sessions.

Demo Prompts

1) "Запомни: проект LinguaSprint, KPI 1500 MAU. Дай 3 риска."
2) "Теперь для этих рисков предложи 5 конкретных мер."
3) "Сделай подробный план из 30 пунктов с пояснениями по каждому."
4) Для overflow: уменьшите `Context limit tokens` и повторите шаг 3.
